# xhbx 启动文档

## 推荐执行命令

正式批处理全库，并开启视觉识别与质检：

```bash
uv run insurance-coach-md run --concurrency 4 --review
```

如需先小批量验证，建议执行：

```bash
uv run insurance-coach-md run --limit 5 --concurrency 2 --review
```

## 项目定位

本项目是保险绩优案例知识沉淀智能体，负责将 `数据/绩优案例/` 下的原始素材
（docx、pptx、pdf、txt）解析、研判并整理为标准化 Markdown 知识单元。

当前边界：

- 产出 Markdown 与元数据 sidecar。
- 不做 embedding。
- 不写入向量库。
- 视觉识别用于抽取 pptx/pdf 配图中的有效信息。
- 质检用于审计 Markdown 规范性、信息保真与无加工旁白。

## 环境要求

- Python 3.12 或更高版本。
- 已安装 `uv`。
- 项目根目录存在 `.env`。
- 原始素材放在 `数据/绩优案例/`。

## 首次启动

在项目根目录执行：

```bash
uv sync
cp .env.example .env
```

然后编辑 `.env`，填入真实模型配置：

```env
QWEN_API_KEY=sk-your-key-here
QWEN_BASE_URL=https://api.mixroute.ai/v1
QWEN_MODEL_NAME=qwen3.7-max
QWEN_VISION_MODEL_NAME=gpt-4o
```

说明：

- `QWEN_MODEL_NAME` 用于文本研判与知识提取。
- `QWEN_VISION_MODEL_NAME` 用于课件配图识别。
- qwen 系列不支持图像输入，因此视觉模型应使用支持图片输入的模型。

## 启动前检查

统计素材库，确认案例、节、文件类型能被扫描到：

```bash
uv run insurance-coach-md stats
```

如果输出提示未发现任何节目录，请检查：

- `数据/绩优案例/` 是否存在。
- 素材是否按“案例 / 节 / 文件”的层级组织。
- 文件类型是否为 docx、pptx、pdf 或 txt。

## 单节调试

预览单个节，不调用模型：

```bash
uv run insurance-coach-md show "<案例>/<节>" --preview 400
```

研判并提取单个节：

```bash
uv run insurance-coach-md build "<案例>/<节>"
```

跳过视觉识别：

```bash
uv run insurance-coach-md build "<案例>/<节>" --no-vision
```

开启质检：

```bash
uv run insurance-coach-md build "<案例>/<节>" --review
```

## 全库批处理

默认按目录分组，将每个“节”作为一个知识单元：

```bash
uv run insurance-coach-md run --concurrency 4
```

开启质检：

```bash
uv run insurance-coach-md run --concurrency 4 --review
```

关闭视觉识别：

```bash
uv run insurance-coach-md run --concurrency 4 --no-vision
```

强制重跑，忽略已有产物：

```bash
uv run insurance-coach-md run --concurrency 4 --force
```

只跑前 5 个知识单元，用于验证配置与成本控制：

```bash
uv run insurance-coach-md run --limit 5 --concurrency 2 --review
```

## 产物位置

批处理完成后，产物写入 `output/`：

```text
output/
├── <案例>/<节>.md
├── <案例>/<节>.meta.json
├── manifest.json
└── .image_cache/
```

关键文件说明：

- `<节>.md`：标准化知识正文，包含 YAML frontmatter。
- `<节>.meta.json`：结构化元数据，包含研判理由、价值评分、主题等。
- `manifest.json`：全库处理汇总，包含每节状态、评分、错误与质检结果。
- `.image_cache/`：课件图片识别缓存。

## 验证命令

运行测试：

```bash
uv run pytest -q
```

测试使用 fake 模型，不调用真实 API。
